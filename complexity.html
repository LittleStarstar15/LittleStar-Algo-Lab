MAIN TERMINAL

理论计算模块：数据量(N)与系统负载

ALGORITHM COMPLEXITY VISUALIZER

* 拖动滑块改变 N，观察系统负载的非线性爆发。

O(1) - 常数阶 0.00ms

例如：数组下标访问。无论 N 多大，耗时恒定。

O(log N) - 对数阶 0.00ms

例如：二分查找。N 翻倍，耗时仅增加一个单位。

O(N) - 线性阶 0.00ms

例如：单层 for 循环。耗时与 N 成正比。

O(N²) - 平方阶 0.00ms

例如：双层嵌套循环。N 增加 10 倍，耗时暴涨 100 倍！

O(1) 空间占用

无论数据规模如何，内存始终维持恒定占用。

O(N) 空间占用

内存消耗随着数据规模 N 的增加而线性增长。
